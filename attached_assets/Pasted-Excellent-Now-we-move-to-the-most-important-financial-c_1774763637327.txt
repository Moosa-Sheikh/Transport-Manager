🔷 STEP 1: DATABASE UPDATES

Open database/schema.sql

Add the following tables.

1️⃣ customer_payments
CREATE TABLE customer_payments (
    id SERIAL PRIMARY KEY,
    trip_id INTEGER REFERENCES trips(id) ON DELETE SET NULL,
    amount NUMERIC(12,2) NOT NULL,
    payment_date DATE NOT NULL,
    payment_mode VARCHAR(50),
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Purpose:
Track money received from factory/company.

2️⃣ driver_advances
CREATE TABLE driver_advances (
    id SERIAL PRIMARY KEY,
    driver_id INTEGER NOT NULL REFERENCES drivers(id) ON DELETE CASCADE,
    trip_id INTEGER REFERENCES trips(id) ON DELETE SET NULL,
    amount NUMERIC(12,2) NOT NULL,
    advance_date DATE NOT NULL,
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Purpose:
Track money given to driver (advance per trip).

3️⃣ driver_salaries
CREATE TABLE driver_salaries (
    id SERIAL PRIMARY KEY,
    driver_id INTEGER NOT NULL REFERENCES drivers(id) ON DELETE CASCADE,
    month VARCHAR(20) NOT NULL,
    year INTEGER NOT NULL,
    amount NUMERIC(12,2) NOT NULL,
    payment_date DATE NOT NULL,
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Purpose:
Fixed monthly salary payment.

4️⃣ cash_book (VERY IMPORTANT)
CREATE TABLE cash_book (
    id SERIAL PRIMARY KEY,
    entry_type VARCHAR(20) NOT NULL, -- IN or OUT
    reference_table VARCHAR(50),
    reference_id INTEGER,
    amount NUMERIC(12,2) NOT NULL,
    entry_date DATE NOT NULL,
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

This will act as:

Company Cash Ledger

Every IN or OUT must create entry here automatically.

Run migration safely.

🔷 STEP 2: PAYMENT ROUTES

Create new route file:

routes/payments.js
A) Add Customer Payment

POST /payments/customer/add

Validation:

amount > 0
date required

Insert into:
customer_payments

Then automatically insert into cash_book:

entry_type = 'IN'
reference_table = 'customer_payments'
reference_id = inserted_id
amount = same
entry_date = payment_date

B) Add Driver Advance

POST /payments/driver-advance/add

Insert into:
driver_advances

Then create cash_book entry:

entry_type = 'OUT'
reference_table = 'driver_advances'
reference_id = inserted_id
amount = same

C) Add Driver Salary

POST /payments/driver-salary/add

Insert into:
driver_salaries

Then cash_book entry:

entry_type = 'OUT'
reference_table = 'driver_salaries'
reference_id = inserted_id

All must use database transaction (BEGIN / COMMIT).

If cash_book insert fails → rollback.
